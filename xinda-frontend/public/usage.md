# 外文档案文献处理工作台 - 使用指南

## ⚠️ 重要说明

本系统的核心功能是调用 AI 模型进行 OCR 识别和翻译，最终效果受以下因素影响：

1. **模型能力**：不同 AI 模型对不同语言的识别和翻译能力存在显著差异
2. **提示词配置**：OCR 识别和翻译的提示词（Prompt）直接影响处理质量
3. **文档质量**：模糊、污损、手写体等文档会降低识别准确率

**建议使用者定期维护模型和优化提示词配置，以获得最佳处理效果。**

---

## 一、系统概述

外文档案文献处理工作台（项目代号"**信达**"）是一款用于处理日文、英文等外语文档的工具。它可以：

- 自动识别 PDF 或图片中的外文文本
- 将识别出的外文翻译成中文
- 支持一键导出识别稿或翻译稿（Word 格式）

---

## 二、首页功能

### 1. 文件上传
- 支持 **PDF 文件** 和 **JPG 图片** 格式
- 点击上传区域或拖拽文件到上传框即可
- 上传成功后会自动显示文件名
- 最大支持 20MB 的文件

### 2. 处理设置
- **识别模型**：选择用于 OCR 识别外文的视觉模型（需要支持图像理解）
- **翻译模型**：选择用于翻译的文本模型
- **语种选择**：支持「自动检测」或手动选择具体语种
  - 建议选择具体语种（如「日文」）以获得更好效果
  - 「自动检测」可能需要额外时间进行语言判断
- **模型设置**：点击可配置模型供应商和模型参数
- **提示词设置**：点击可调整 OCR 识别和翻译的提示词模板

### 3. 开始处理
- 选择模型后，点击「开始处理」按钮
- 系统会跳转到结果页，后台开始处理
- 处理是异步的，可以在处理过程中切换页面

### 4. 处理历史
- 页面下方显示历史处理记录
- 点击记录可查看之前处理的结果
- 已完成的记录可直接打开查看

---

## 三、结果页功能

### 1. 页面布局
- 左侧：原始文档预览
- 中间：外文识别结果（OCR）
- 右侧：中文翻译结果

### 2. 文档操作
- **翻页**：使用「上一页/下一页」按钮，或直接点击页码输入跳转
- **缩放**：使用「＋/－/还原」按钮调整显示比例
- **拖拽**：支持鼠标拖动查看大图

### 3. 模型与提示词
- 点击「模型设置」可修改 OCR/翻译模型
- 点击「提示词设置」可调整识别和翻译的提示词
- 修改后只会影响新开始的处理，不会改变已保存的结果

### 4. 重新处理
- **清空重新开始**：清空所有结果，从第一页重新开始处理
- **继续未完成处理**：从中断页继续处理（如果有页面未完成）
- **单页重新处理**：点击任意页的刷新按钮，可单独重新处理该页

### 5. 结果导出
- **识别稿导出**：仅导出 OCR 识别结果（外文原文）
- **翻译稿导出**：仅导出翻译结果（中文）
- 导出会生成 Word 文档，方便后续编辑

### 6. 状态说明
| 状态 | 说明 |
|------|------|
| 等待中 | 等待处理 |
| 处理中 | 正在识别和翻译 |
| 处理完成 | 全部完成 |
| 处理失败 | 处理出错 |

## 四、提示词设置说明

提示词设置是影响识别和翻译质量的关键因素，包含 5 个标签页：

### 4.1 提示词类型

| 标签页 | 用途 |
|--------|------|
| OCR 日文 | 日文文档的识别提示词 |
| OCR 英文 | 英文文档的识别提示词 |
| 翻译 日文 | 日文翻译成中文的提示词 |
| 翻译 英文 | 英文翻译成中文的提示词 |
| 语言检测 | 自动检测文档语种的提示词 |

### 4.2 提示词编写建议

**OCR 识别提示词示例：**
> 请仔细识别图片中的所有文字，包括标题、正文和注释。保持原有格式，只输出纯文本内容。

**翻译提示词示例：**
> 请将以下日文内容翻译成中文，保持原文的语气和专业术语的翻译准确性。

### 4.3 维护建议

1. **分语种配置**：为每种常用语言设置专属提示词
2. **根据效果调整**：如果识别/翻译效果不理想，尝试修改模型与提示词
3. **参考模型文档**：不同模型可能对提示词格式有不同响应
4. **保存配置**：修改提示词后点击保存，系统会自动应用新提示词

## 五、模型配置说明

### 5.1 添加模型供应商

在「模型设置」中可以添加新的 AI 供应商：

1. 点击「添加供应商」
2. 填写供应商信息：
   - 名称：如 "Ollama"、"OpenAI"、"Custom AI"
   - 显示名称：用于界面显示的名称
   - API 地址：模型的 API 端点
   - API Key：访问密钥（如需要）
3. 添加可用模型列表
4. 测试连接确保配置正确

### 5.2 推荐模型

**OCR 识别（视觉/多模态模型）：**

| 模型 | 供应商 | 推荐理由 |
|------|--------|----------|
| **Qwen2.5-VL** | Ollama/OpenAI | 多语言支持优秀，特别是亚洲语言（日韩中文），推荐用于日文档案 |
| **Llama 3.2 Vision** | Ollama | 开源免费，本地运行，英文档案效果较好 |
| **GPT-4o** | OpenAI | 性能最强，综合效果最好 |
| **Gemini 2.5 Flash** | Google | 速度快，性价比高，文档理解能力强 |
| **Claude 4.1 Opus** | Anthropic | 推理能力强，技术文档分析出色 |

**翻译（文本模型）：**

| 模型 | 供应商 | 推荐理由 |
|------|--------|----------|
| **Qwen2.5** | Ollama | 多语言翻译效果好，开源免费 |
| **Llama 3.1** | Ollama | 开源免费 |
| **GPT-4o** | OpenAI | 翻译质量高 |
| **Gemini 2.5 Pro** | Google | 支持长上下文 |

**按语种选择建议：**
- **日文档案**：Qwen2.5-VL（识别） + Qwen2.5（翻译）
- **英文档案**：Llama 3.2 Vision 或 GPT-4o
- **德文/法文档案**：GPT-4o 或 Gemini 2.5
- **混合语种档案**：GPT-4o 或 Qwen2.5-VL

## 六、常见问题

**Q：上传文件大小有限制吗？**
A：最大支持 20MB 的文件。

**Q：处理时间很长怎么办？**
A：
- 页面会实时显示处理进度
- 可以点击进度条右侧的刷新按钮重新处理当前页
- 检查选择的模型性能是否足够

**Q：识别结果有错误怎么办？**
A：
- 可以直接在结果页点击对应页的刷新按钮
- 使用调整后的模型或提示词重新处理该页
- 建议在「提示词设置」中优化提示词

**Q：翻译质量不好怎么办？**
A：
- 尝试使用更强大的翻译模型
- 在「提示词设置」中优化翻译提示词
- 可以针对特定语言调整专属提示词

**Q：自动检测语种不准确怎么办？**
A：建议手动选择具体语种（而非「自动检测」），可获得更稳定的效果。

---

## 七、维护与优化建议

1. **定期更新模型**：使用更强大的模型版本可显著提升效果
2. **优化提示词**：根据实际处理结果持续调整提示词，这是提升质量最有效的方式
3. **分语种配置**：为每种常用语言（如日文、英文、德文、法文）设置专属提示词
4. **监控处理效果**：定期检查识别和翻译质量，及时调整配置
